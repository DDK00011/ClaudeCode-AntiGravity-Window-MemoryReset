# 작업 일지 — 2026-04-19 (v1.1.x: Deep + Diagnose)

> v1.0 (95%→31% 메모리 회수 검증) 위에 추가된 v1.1 / v1.1.1 세션 기록.
> Round 1 셀프 리뷰 + Round 2 독립 에이전트 리뷰 전 과정.

## 동기

v1.0 출시 후 사용자 피드백:
- **현상**: Antigravity + Claude Code 재기동 후 PC 가 "재부팅보다는 덜 쾌적함"
- **가설**: 본 스크립트가 회수하지 않는 다른 캐시(Memory Compression Store, kernel pool, GPU 드라이버, explorer 셸 캐시 등) 누적이 원인

검증 결과: **가설 정확**. v1.0 의 4단계 (working set / file cache / modified list / standby purge) 외에도 누적되는 영역 다수 확인.

## 추가된 기능 (v1.1.0 → v1.1.1)

### Tier A — `-Deep` (안전, 권장)
- **Memory Compression Store flush**: `Disable-MMAgent -MemoryCompression` → 1500ms wait → `Enable-MMAgent`. 누적 1~3 GB 해제
- **System Working Set empty (NT 레벨)**: `NtSetSystemInformation` command 2. per-process 루프가 `PROCESS_SET_QUOTA` 권한 부족으로 skip 한 보호 프로세스를 커널 모드에서 보완
- **DNS / NetBIOS / ARP 캐시 정리**: `Clear-DnsClientCache` / `nbtstat -R` / `netsh interface ip delete arpcache`

### Tier B — `-IncludeShell` (옵션, 깜빡임 동반)
- **Explorer.exe 재시작**: 셸 캐시 200~500 MB 해제. Windows 자동 재시작에만 의존 (elevation 위험 회피)
- **Windows Search 서비스 재시작**: 인덱서 누적 메모리 해제

### `-Diagnose` 모드 (read-only, UAC 불필요)
- PerfCounter 기반 메모리 리스트 분포 (Standby Cache by priority / Modified / Free & Zero / Cache)
- Memory Compression Store 상태
- 상위 15 점유 프로세스 (Working Set + Commit charge)
- 종료 대상 프로세스 미리보기

### Tier C — **의도적으로 제외**
디스플레이 어댑터 disable/enable (GPU 메모리 reset). 이유:
- 게임/영상편집/녹화/스트리밍 작업 중 데이터 손실 위험
- 멀티모니터 재배치 필요
- DirectX/OpenGL 활성 앱 crash 가능

대안: README 에 `Win+Ctrl+Shift+B` 수동 단축키 안내

## 다단계 검토 결과

### Round 1 (셀프 리뷰) — 4건 발견, 모두 패치

| # | 심각도 | 이슈 | 패치 |
|---|--------|------|------|
| 1 | P1 | `Disable-MMAgent` 후 `Enable-MMAgent` 실패 시 시스템 압축 영구 비활성화 | try/finally + dual-failure CRITICAL 알림 |
| 2 | P1 | Explorer 재시작 1500ms 고정 대기 (셸 확장 많은 시스템 부족) | 폴링 루프 도입 |
| 3 | P1 | 메모리 95% 상태에서 압축 flush 시 decompress spike 로 OOM 위험 | 가용 RAM < 1024 MB 면 skip |
| 4 | P2 | "Memory Compression" 프로세스 정보 메시지 모호 | admin/build 케이스 구분 |

### Round 2 (독립 에이전트 리뷰) — 추가 P1 4건, 핵심 2건 패치

| # | 심각도 | 이슈 | 결정 |
|---|--------|------|------|
| 1 | P1 | PerfCounter 영문 경로 한국어 Windows 미동작 | **유지** — 실측 동작 확인 (Vista+ 자동 매핑) |
| 2 | P1 | `Get-Process 'Memory Compression'` dead code | **유지** — graceful fallback OK, 메시지만 개선 |
| 3 | P1 | NT command 2 와 per-process 루프 관계 불명확 | **패치** — 주석 추가 |
| 4 | P1 | **Explorer 수동 시작이 elevated 컨텍스트로 시작** → 일반 앱 권한 부족 | **패치** — 수동 시작 제거, 사용자 가이드 표시 |

추가 적용:
- MMAgent decompress sleep 800ms → 1500ms (GB-scale store 대응)
- NT command 2 의 보완 효과 주석 명확화

## 적용 안 한 항목 (의도적)

- **PerfCounter ID-based fallback**: 추가 복잡도 vs 실측 동작 정상 → 유지
- **Get-Process compression 분기 제거**: 일부 빌드에서는 동작할 수 있으므로 graceful fallback 유지
- **Tier C (GPU 어댑터 reset)**: 위험성 너무 큼
