## 설계 핵심

### 1. 트레이 데몬 = 비관리자 + 회수 트리거 = 관리자
- **데몬 자체**: 비관리자 (UAC 안 뜸, 부팅 시 자연 시작)
- **회수 액션 클릭**: `MemoryReset.ps1` spawn → 그 안에서 UAC 승격
- 이 분리로 부팅 시 매번 UAC 안 뜨고도 회수 시 보안 가시성 유지

### 2. 알림 정책
- 메모리 사용률 ≥ 90% (사용자 조정 가능, 50~99) → BalloonTip
- **자동 회수 안 함** (사용자 결정 보장)
- 쿨다운 10분 (스팸 방지)

### 3. CSV 스키마 (v1.2.1 명확화)
```
Timestamp, Mode, TotalMB,
UsedBeforeMB, UsedAfterMB, FreedMB,         ← 양수 = 회수 성공
BeforeFreeMB, AfterFreeMB,
BeforePctFree, AfterPctFree, FreedPctP,     ← 양수 = 가용 % 증가
ProcessesKilled, RuntimeSec
```

### 적용한 P1 패치

#### 1. STA 스레드 명시 (Tray.bat)
- **문제**: PS 5.1 기본 ApartmentState = MTA, 그러나 WinForms 는 STA 필수
- **결과**: InputBox / OpenFileDialog 같은 OLE-backed 컨트롤이 hang 가능
- **수정**: `Tray.bat` 에 `-Sta` 플래그 추가
- **출처**: [MS Docs STA](https://learn.microsoft.com/en-us/dotnet/api/system.stathreadattribute)

#### 2. CSV 컬럼명 명확화
- **문제**: `RecoveredMB` 가 의미 혼동
  - 코드는 `AfterFree - BeforeFree` (가용 메모리 증가량)
  - 자연어는 "회수된 메모리 = 사용량 감소"
- **수정**: 스키마 확장
  - `TotalMB` 추가
  - `UsedBeforeMB`, `UsedAfterMB` 추가
  - `RecoveredMB` → `FreedMB` (= UsedBefore - UsedAfter)
  - `RecoveredPctP` → `FreedPctP`
- **호환성**: 컬럼 추가/이름 변경 — 기존 CSV 와 헤더 다름. 새 파일로 시작 권장.

#### 3. Fallback CSV 동시초 충돌
- **문제**: `recovery-history-yyyyMMdd-HHmmss.csv` 가 동일 초에 두 번 생성 시 `Export-Csv` (no -Append) 가 덮어씀
- **수정**:
  - 일자별 단일 fallback: `recovery-history-fallback-yyyyMMdd.csv` + `-Append`
  - 비상 시 (fallback 락도): `recovery-history-emergency-yyyyMMdd-HHmmss-fff-{4자GUID}.csv` (충돌 0%)

### 검증된 사항 (외부 출처)

- **Application.Run 메시지 루프**: NotifyIcon.Visible=$true 가 Run() 전에 OK ([MS Docs](https://learn.microsoft.com/en-us/dotnet/api/system.windows.forms.notifyicon.visible))
- **InputBox 모달 차단**: 동일 STA 스레드에서 메시지 펌핑 → 트레이 freeze 안 함 ([MS Docs](https://learn.microsoft.com/en-us/dotnet/api/microsoft.visualbasic.interaction.inputbox))
- **NotifyIcon.Text 63자 제한**: NOTIFYICONDATA.szTip Win32 API 제한 정확
- **Export-Csv -Append PS 5.1**: 헤더 중복 회피 정상
- **mutex `Local\`**: 사용자 세션 격리 + RDP/Fast User Switching 시 독립 인스턴스 (의도)
